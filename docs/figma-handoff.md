# Rovik Figma Handoff

## Intent

Create a single premium scrolling frame that mirrors the coded page and tells an investor-ready startup story. The page should feel warm, confident, and future-facing rather than generic SaaS or sci-fi dark mode.

## Visual direction

- Background: soft parchment-to-mint gradient with subtle atmospheric glow
- Tone: premium, optimistic, operational
- Shapes: large rounded panels, circular glows, soft borders
- Motion cues in code: floating orbs and subtle drift, not heavy animation

## Typography

- Display: Space Grotesk or closest available geometric grotesk
- Body: Manrope or closest available humanist sans
- Utility / labels: IBM Plex Mono or closest available mono

## Color tokens

- Ink: `#10212F`
- Background: `#F6EFE4`
- Surface: `rgba(255,255,255,0.72)`
- Muted text: `#5A6874`
- Teal accent: `#0F8A7B`
- Deep teal: `#14695F`
- Soft mint: `#DFF8F2`
- Warm accent: `#FFB257`

## Section order

1. Sticky top bar with Rovik brand, section anchors, and primary CTA
2. Hero with one-sentence pitch, metrics, and a command-center preview
3. Problem section contrasting current assistants with Rovik
4. Solution principles section
5. Four capability workflow cards
6. System architecture panel
7. Market opportunity and competitive advantage split section
8. Business model and funding split section
9. Final vision / elevator pitch CTA block

## Figma primitives to create

- Section heading
- Metric card
- Preview card
- Principle card
- Workflow card
- Architecture node
- Architecture lane card
- Pricing card
- CTA button

## Suggested frame setup

- Desktop frame width: `1440`
- Mobile validation frame width: `390`
- Use an 8px spacing system
- Keep panel radii between `24` and `40`
- Keep content centered inside a max-width equivalent of roughly `1280`

## Notes

- The command-center hero should stay abstract and product-like without inventing fake hardware renders.
- Reuse the same surface, border, and shadow language across cards so the page feels like one system.
- If the coded page changes, update this file first, then sync the Figma frame from it.
